# 2026-02-11

## OpenClaw（100.40.1.115 / csit）— UI 只顯示「...」不回覆的修復

### 症狀
- Control UI（http://127.0.0.1:18789）送訊息後只出現「...」然後停住。
- Gateway 端可收到訊息並觸發 embedded run，但 session 記錄中 assistant 回覆 `content: []`。
- `~/.openclaw/agents/main/sessions/<session>.jsonl` 顯示大量：
  - `stopReason: "error"`
  - `errorMessage: "fetch failed"`
- Node.js 測試：`fetch('https://api.openai.com/v1/models')` 失敗，原因：
  - `UNABLE_TO_GET_ISSUER_CERT_LOCALLY` / `unable to get local issuer certificate`

### 根因
- macOS 上 Node（undici/fetch）在該環境無法完成 TLS 憑證鏈驗證（缺少可用的 root/intermediate 信任鏈），導致 OpenClaw 呼叫模型 API 全部失敗，UI 只剩 loading。

### 修復方法（採用）
1) 匯出 macOS System Root Certificates 成 PEM：
   - `security find-certificate -a -p /System/Library/Keychains/SystemRootCertificates.keychain > ~/.openclaw/certs/system-roots.pem`
   - 權限：`chmod 600 ~/.openclaw/certs/system-roots.pem`
2) 讓 Gateway service 的 Node 使用該 CA bundle：
   - 在 `~/Library/LaunchAgents/ai.openclaw.gateway.plist` 的 `EnvironmentVariables` 新增：
     - `NODE_EXTRA_CA_CERTS=/Users/csit/.openclaw/certs/system-roots.pem`
3) 重啟 Gateway（LaunchAgent），使 env 生效。

### 驗證
- 在 shell 中設定 `NODE_EXTRA_CA_CERTS` 後，Node fetch 可回到 `401 Missing bearer authentication`（代表 TLS 已 OK）。
- 重啟 gateway 後，Control UI 送訊息即可正常回覆。

## SSH 非互動環境找不到 openclaw 的修復
- 問題：`ssh csit@100.40.1.115 openclaw ...` 會 `command not found`（non-interactive zsh PATH 太短）。
- 解法：
  - 建立 `/usr/local/bin/openclaw` wrapper，固定用 `/usr/local/bin/node` 執行 openclaw.mjs。
  - 在 `~/.zshenv` 加入：確保 `/usr/local/bin` 在 PATH（對 non-interactive 生效）。
